02. Sign and Verify a Message

Sign and Verify

Note: In this section, we will use the phrase "transaction" and "message" interchangeably. A "message" is often part of a "transaction" or can be a form of a simple "transaction" of communication.

## Why sign and verify messages?

A Bitcoin digital signature and its verification is one of the main key secrets behind the Bitcoin protocol.

We have already discussed the importance of "Signing a Transaction".

This process allows non-repudiation which means the person who sent the message had to be in possession of the private key in order to send the message, and as a result, anyone on the network can use this information to verify the transaction.

But how does it work?

  • The sender generates a private key and a public key, using a wallet address.
  • The sender then signs the message with the signature, and sends their public key, the signature and the message to the network.
  • The receiving node then checks using the verification algorithm that the message has been signed by the sender, which can only be done by the holder of the private key to the public key that is sent.
  • Signing a transaction helps to prove ownership of these transactions and to prove that transaction and blockchain identity were not tampered with.

## 3 Step Process

At a high-level, there are 3 steps in the process:

Step 1 Generate a private key and a public key, using a wallet address
Step 2 Sign a message using the wallet address
Step 3 Verify the message using the address and the signature
