11. Testnet Wallet Transactions

We have made this lesson, assuming that have downloaded Bitcoin-Core v.016, from here https://bitcoincore.org/bin/bitcoin-core-0.16.3/. We recommend you to use this version, to follow along these and further lessons.

This is the latest version available while making this lesson.

If you are using v0.17, we have added new instructions for you, so skip this concept and go to the next one "Bitcoin Core v0.17: Wallet Creation and Coin Transfer" and follow those instructions.

Testnet Wallet Transactions

Transactions are a fundamental component of the entire bitcoin blockchain. It’s important to understand how to make and manage transactions using your wallet. Generally, you’d make transactions with the owner of another wallet.

To start testing transactions, you’ll need to make these same transactions on your local computer. To do this, you’ll need multiple wallets that allow you to jump back and forth to check out the transactions your making.

Here, we’ll learn how to do exactly that.

We’ll create multiple wallets then start to send transactions between them.

Upcoming Steps

To do this we’ll go through a few steps…

  • Step 1: Find your Wallet
  • Step 2: Create New Wallets
  • Step 3: Set up Bitcoin.config file
  • Step 4: Open Receiving Wallet
  • Step 5: Get Receiving Address
  • Step 6: Switch to sending Wallet
  • Step 7: Get Testnet Coins

  • Step 9: Check Transaction in Receiving Wallet

This is a lot of steps, but it’s a great way to send transactions among multiple wallets that you own. This will allow you to pass coins between wallets both now, and later programmatically when we start building applications.

Step 1: Find your Wallet

From the user interface, we can’t actually create and jump between multiple wallets. What we need to do is create new wallets out of the files that store our application information.

  • Navigate into the folder that contains all of your bitcoin core files.

  • Within the main folder, you’ll see one labeled testnet3.

  • Go into this folder and look for the sub-folder labeled wallets.

What you should see is a file named wallet.dat. This is the main file that stores all of the information for your wallet.

Step 2: Create New Wallets

To easily create new test wallets, you can duplicate this file. (Right click + Duplicate)

  • For this example, duplicate this wallet 2 times.
  • Rename one of the copies sending.dat
  • Name the other copy receiving.dat.

As the names suggest, we’ll use the sending wallet to send test coins to the receiving wallet with our transaction.

Step 3: Set up Bitcoin Config File

Before we move on, it’s important to know how to get into each of these wallets. By default, the bitcoin core application will look for the wallet.dat file to load along with the application.

One way to open a wallet is to rename the one you want opened to wallet.dat before you load the application. While this is possible, it’s not really the best option.

What I prefer is to change the name of what the application is looking for. Many default options that you might like to override can be changed using the bitcoin config file.

This is the same file we use to change between networks.

  • To make this change, open the bitcoin.config file and type wallet = .

Next, type in the wallet name you want loaded.

Once you save the file this is will be the default wallet for your application. One thing to note is that if you type in a name here that doesn’t exist, a wallet with that name will be created when you first load the application.

Ok, knowing that, let’s type in both of our wallet names.

  • wallet = sending.dat
  • wallet = receiving.dat

Step 4: Open Receiving Wallet

We’ll start by opening the receiving wallet, so for now, comment out the sending.dat line with a #.

  • #wallet = sending.dat

Once you’re in, you can check the wallet name by going to the debug console and typing in getwalletinfo.

This shows us all the wallet information, and most importantly for right now, we can see that our wallet name is receiving.dat.

Step 5: Get Receiving Address

We’re only going to use this wallet to receive money from our sending wallet.

To do this we need a receiving address.

  • To get one Go to file > receiving addresses > new address
  • Then copy the address and put it somewhere like a text document.

Step 6: Switch to Sending Wallet

Now we want to switch over to our sending wallet.

  • In the bitcoin config file, remove the comment from the sending wallet and add it to the receiving wallet.
  • Now restart the application again.

  • wallet = sending.dat
  • #wallet = receiving.dat

Once inside the wallet, head back to the debug console and check the name of the wallet using getwalletinfo.

Looks like we’re in the sending wallet just like we wanted.

The problem now is that we don’t have any coins to send.

To get some, we can go back to the public faucet and request coins like we did before.

Step 7: Get testnet coins

  • To get testnet coins go to Testnet Faucet

  • Create and copy a receiving address for this wallet and request coins from the faucet.

  • Soon you should have some testcoins that we can send over to our receiving wallet.

  • You can check the overview tab to see both the transaction and the new balance.

After all of this, we’re finally ready to make a transaction.

Step 8: Make a Transaction

  • Go to the send tab and paste in the receiving address you had stored from your receiving wallet.

  • Add in a label, an amount, and a transaction fee if you’d like.

Once you hit send, the transaction will begin.

Back on the overview page, you’ll see a new recent transaction for the amount you specified.

You’ll also see that the available balance has decreased.

Step 9: See Transaction on Receiving Wallet

The transaction is done, and we could stop here, but it’ll be nice to see if the transaction has been received by the receiving wallet.

To do this, we need to switch wallets just like before.

  • Make the small changes to the bitcoin config file and restart the application.

Now, in the overview of your receiving wallet, you should see that the balance has gone up, and that there’s been a recent transaction for the amount you specified.

If you didn’t add in the transaction fee up front, you’ll notice that the amount you received is actually slightly less than what you might expect. That’s because the transaction fee was taken out of the total amount you sent.

Wrapping Up

That’s how to create transactions using your bitcoin core wallet. What we did here is actually WAY more than create a transaction. Making the transaction was really the simple part of all this. All it takes to do this is to put a receiving address, and an amount using the send tab.

All the other steps we did were mostly to prepare us for testing and help us make transactions with ourselves. What we really did was set up 2 extra test wallets, got some coins from a faucet, and send those coins from one wallet to another.

We’ll continue to use these test wallets as we build and test applications in the future.
